Objective

Implement a Digital Signature System using Python to understand public-key cryptography. Generate an RSA key pair, sign a message using the private key, and verify the signature using the public key. SHA-256 hashing ensures message integrity before signing.

How It Works

1
Key Generation: An RSA 2048-bit key pair (private + public) is generated using Python's cryptography library.
2
Hashing: The message is hashed using SHA-256 to create a fixed-size digest, ensuring integrity.
3
Signing: The private key signs the message digest using RSA-PSS padding, creating the digital signature.
4
Verification: The public key verifies the signature against the original message. A mismatch means tampering.

Source Code

digital_signature.py PYTHON 
# ALA 1: Digital Signature Verifier
# Implements RSA key pair generation, signing, and verification

from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.backends import default_backend
from cryptography.exceptions import InvalidSignature
import base64

# ─── Step 1: Generate RSA Key Pair ───────────────────────────────────────────
def generate_keys():
    private_key = rsa.generate_private_key(
        public_exponent=65537,
        key_size=2048,
        backend=default_backend()
    )
    public_key = private_key.public_key()
    return private_key, public_key

# ─── Step 2: Sign a Message ───────────────────────────────────────────────────
def sign_message(private_key, message: str):
    message_bytes = message.encode('utf-8')
    signature = private_key.sign(
        message_bytes,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        ),
        hashes.SHA256()
    )
    return signature

# ─── Step 3: Verify the Signature ────────────────────────────────────────────
def verify_signature(public_key, message: str, signature):
    message_bytes = message.encode('utf-8')
    try:
        public_key.verify(
            signature,
            message_bytes,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()),
                salt_length=padding.PSS.MAX_LENGTH
            ),
            hashes.SHA256()
        )
        return True
    except InvalidSignature:
        return False

# ─── Main Execution ───────────────────────────────────────────────────────────
if __name__ == "__main__":
    message = "Hello, this is a secure message from the sender!"

    print("=== Digital Signature Verifier ===")
    print(f"\n[*] Original Message : {message}")

    # Generate keys
    private_key, public_key = generate_keys()
    print("\n[+] RSA Key Pair Generated (2048-bit)")

    # Sign
    signature = sign_message(private_key, message)
    sig_b64 = base64.b64encode(signature).decode()
    print(f"\n[+] Signature (base64):\n    {sig_b64[:64]}...")

    # Verify original message
    result = verify_signature(public_key, message, signature)
    print(f"\n[✓] Signature Valid for original message : {result}")

    # Tamper test
    tampered = "Hello, this is a TAMPERED message!"
    result_tampered = verify_signature(public_key, tampered, signature)
    print(f"[✗] Signature Valid for tampered message  : {result_tampered}")

Analysis

🔐 Why RSA?

RSA is an asymmetric algorithm — what the private key signs, only the public key can verify. This enables non-repudiation and authentication without sharing secret keys.

#️⃣ Role of SHA-256

Hashing compresses any message to a fixed 256-bit digest. Signing the hash (not the full message) is faster and the hash ensures any modification to the message is detectable.

🧪 Tamper Detection

When the tampered message is verified, the computed hash doesn't match the signed hash, so verification fails — proving the system correctly detects data tampering.

📦 PSS Padding

RSA-PSS (Probabilistic Signature Scheme) adds randomized salt to prevent deterministic attacks. Each signature of the same message produces a different output.
